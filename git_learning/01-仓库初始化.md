# 01 仓库初始化

这一节只解决一个问题：怎样把一个普通文件夹变成 Git 仓库。

## Git 仓库是什么

普通文件夹只保存“当前文件”。

Git 仓库除了保存当前文件，还保存文件的历史版本。也就是说，Git 可以帮你记录：

- 哪些文件被修改了
- 每次修改了什么
- 谁在什么时候提交了修改
- 如果改错了，能不能回到之前的版本

## 初始化仓库

进入你的项目目录后，执行：

```bash
git init
```

执行后，当前文件夹下面会多出一个隐藏目录：

```text
.git/
```

这个 `.git/` 文件夹就是 Git 的核心。它里面保存了版本历史、分支信息、提交记录等内容。

## 判断当前目录是不是 Git 仓库

可以执行：

```bash
git status
```

如果当前目录是 Git 仓库，会看到类似信息：

```text
On branch main
No commits yet
```

如果不是 Git 仓库，可能会看到：

```text
fatal: not a git repository
```

## 第一次配置用户名和邮箱

Git 需要知道是谁提交了代码。

全局配置：

```bash
git config --global user.name "你的名字"
git config --global user.email "你的邮箱"
```

查看配置：

```bash
git config --global --list
```

## 初始化后的第一件事

通常先创建或检查 `.gitignore`。

`.gitignore` 用来告诉 Git：哪些文件不要提交。

比如：

```gitignore
.venv/
__pycache__/
.idea/
*.pkl
data/
```

这些通常是本地环境、缓存、大文件或临时结果，不适合上传 GitHub。

## 本节练习

新建一个测试文件夹，然后执行：

```bash
git init
git status
```

观察：

- 是否出现 `.git/`
- `git status` 输出了什么

## 初学者要记住

`git init` 只需要在一个项目里执行一次。

以后再进入这个项目，不需要重复初始化，直接使用 `git status` 查看状态即可。
