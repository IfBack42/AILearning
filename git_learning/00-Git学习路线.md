# Git 学习路线

这个文件夹是我给自己整理的 Git 学习路线。目标不是背命令，而是知道：什么时候该用哪个命令、这个命令解决什么问题、命令执行后仓库发生了什么变化。

## 为什么 Git 学起来会烦躁

Git 容易让初学者烦躁，不是因为它真的特别难，而是因为它一次性丢给你很多概念：

- 工作区
- 暂存区
- 本地仓库
- 远程仓库
- 分支
- 合并
- 冲突
- 推送
- 拉取
- Pull Request

如果这些概念混在一起学，就会觉得命令很多、状态很多、错误也很多。

所以我把学习顺序拆开：先学本地，再学远程；先学单人，再学协作。

## 推荐学习顺序

### 第一阶段：知道 Git 是干什么的

阅读：

- `01-仓库初始化.md`

你需要理解：

- Git 是代码版本管理工具
- 仓库就是 Git 管理的项目文件夹
- `git init` 会把普通文件夹变成 Git 仓库
- `.git` 文件夹保存 Git 的版本信息

### 第二阶段：掌握本地最常用命令

阅读：

- `02-本地常用命令.md`

你需要会用：

- `git status`
- `git add`
- `git commit`
- `git log`
- `git diff`

这一阶段最重要，因为你以后每天都会用这些命令。

### 第三阶段：理解分支

阅读：

- `03-分支管理.md`

你需要理解：

- 分支是用来隔离开发任务的
- 主分支一般叫 `main`
- 新功能最好在新分支开发
- 开发完成后再合并回主分支

### 第四阶段：连接 GitHub

阅读：

- `04-远程仓库和GitHub.md`

你需要会用：

- `git remote`
- `git push`
- `git pull`
- `git clone`

这一阶段开始，你的代码就不只是保存在电脑上，也可以同步到 GitHub。

### 第五阶段：学习合作流程

阅读：

- `05-团队协作和PR.md`

你需要理解：

- Pull Request 是代码合并申请
- Code Review 是别人检查你的代码
- 冲突不是错误，而是 Git 不知道该保留哪份修改

### 第六阶段：遇到问题时查表

阅读：

- `06-常见问题速查.md`

你需要知道：

- 写错 commit message 怎么办
- add 错文件怎么办
- commit 后又想修改怎么办
- pull 出冲突怎么办

## 最小可用命令表

如果只记最少的命令，可以先记下面这些：

```bash
git status
git add .
git commit -m "说明这次改了什么"
git log --oneline
git branch
git switch -c 分支名
git switch main
git push
git pull
```

## 我的学习建议

不要先背命令参数。

先按照这个顺序练：

1. 新建一个测试文件夹
2. `git init`
3. 新建一个文件
4. `git status`
5. `git add`
6. `git commit`
7. 修改文件
8. 再看 `git status` 和 `git diff`
9. 再提交一次
10. 看 `git log`

Git 真正入门的标志不是背了多少命令，而是你能看懂 `git status` 告诉你的当前状态。
