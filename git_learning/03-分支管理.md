# 03 分支管理

这一节学习 Git 里非常重要的概念：分支。

## 分支是什么

可以把分支理解成一条独立的开发线。

比如：

```text
main       主分支，保存稳定代码
feature    新功能分支，用来开发新功能
fix         修复问题的分支
```

你在新分支上写代码，不会马上影响主分支。等功能完成后，再把它合并回主分支。

## 查看当前分支

```bash
git branch
```

当前所在分支前面会有一个 `*`。

## 创建并切换分支

```bash
git switch -c 分支名
```

例如：

```bash
git switch -c feature/readme
```

意思是：创建一个叫 `feature/readme` 的分支，并切换过去。

## 切换已有分支

```bash
git switch main
```

## 合并分支

假设你在 `feature/readme` 分支完成了 README 修改。

先切回主分支：

```bash
git switch main
```

再合并：

```bash
git merge feature/readme
```

## 删除已经合并的分支

```bash
git branch -d feature/readme
```

注意：删除分支不等于删除代码。如果分支已经合并，代码还在主分支里。

## 常见分支命名

```text
feature/readme      新功能
fix/gitignore       修复问题
docs/git-learning   文档修改
test/model-demo     实验代码
```

分支名最好能看出你在做什么。

## 本节练习

```bash
git switch -c docs/git-learning
git branch
```

然后修改一个文档文件，提交：

```bash
git add .
git commit -m "添加 Git 分支学习笔记"
```

切回主分支并合并：

```bash
git switch main
git merge docs/git-learning
```

## 初学者要记住

分支不是高级技巧，而是日常开发习惯。

以后做新功能、改文档、修 bug，都可以先开一个新分支。
