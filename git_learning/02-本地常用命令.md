# 02 本地常用命令

这一节学习 Git 最核心、最常用的一组命令。

## Git 的三个区域

初学 Git 一定要先理解三个区域：

```text
工作区 -> 暂存区 -> 本地仓库
```

### 工作区

你正在编辑的项目文件夹。

比如你修改了一个 `.py` 文件，这个修改首先发生在工作区。

### 暂存区

准备提交的文件清单。

执行 `git add` 后，文件会进入暂存区。

### 本地仓库

真正保存版本记录的地方。

执行 `git commit` 后，暂存区里的内容会保存成一次提交。

## 查看状态

```bash
git status
```

这是最重要的命令之一。

它会告诉你：

- 哪些文件被修改了
- 哪些文件还没被 Git 跟踪
- 哪些文件已经加入暂存区
- 当前在哪个分支

遇到不知道下一步做什么时，先运行：

```bash
git status
```

## 添加到暂存区

添加某个文件：

```bash
git add 文件名
```

添加当前目录下所有改动：

```bash
git add .
```

初学时可以常用 `git add .`，但提交前一定要先看 `git status`，避免把不该提交的大文件、临时文件也加进去。

## 提交版本

```bash
git commit -m "提交说明"
```

例如：

```bash
git commit -m "添加 Git 学习笔记"
```

提交说明要写“这次做了什么”，不要写成：

```text
update
111
fix
```

更好的写法：

```text
添加 README 项目介绍
整理 Python 基础练习目录
忽略虚拟环境和模型文件
```

## 查看提交历史

```bash
git log
```

简洁版本：

```bash
git log --oneline
```

这个命令可以看到每一次 commit。

在 git log 的分页查看器中，可以：
```bash
q - 退出
空格键 - 向下翻一页
b - 向上翻一页
↑ / ↓ - 逐行上下滚动
/关键词 - 搜索内容（按 n 查找下一个）
g - 跳到开头
G - 跳到末尾
```

## 查看修改内容

查看工作区中还没有暂存的修改：

```bash
git diff
```

查看已经暂存、准备提交的修改：

```bash
git diff --cached
```

## 一次完整流程

```bash
git status
git add .
git status
git commit -m "添加 Git 学习笔记"
git log --oneline
```

这就是最基础的一轮 Git 操作。

## 本节练习

1. 新建一个 `hello.txt`
2. 写入一句话
3. 执行 `git status`
4. 执行 `git add hello.txt`
5. 再执行 `git status`
6. 执行 `git commit -m "添加 hello 文件"`
7. 执行 `git log --oneline`

## 初学者要记住

`git add` 不是提交。

`git commit` 才是真正保存一个版本。
