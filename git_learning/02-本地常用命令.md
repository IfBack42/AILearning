# 02 本地常用命令

这一节学习 Git 最核心、最常用的一组命令。

## Git 的三个区域

初学 Git 一定要先理解三个区域：

```text
工作区 -> 暂存区 -> 本地仓库
```

### 工作区

你正在编辑的项目文件夹。

比如你修改了一个 `.py` 文件，这个修改首先发生在工作区。

### 暂存区

准备提交的文件清单。

执行 `git add` 后，文件会进入暂存区。

### 本地仓库

真正保存版本记录的地方。

执行 `git commit` 后，暂存区里的内容会保存成一次提交。

## 查看状态

```bash
git status
```

这是最重要的命令之一。

它会告诉你：

- 哪些文件被修改了
- 哪些文件还没被 Git 跟踪
- 哪些文件已经加入暂存区
- 当前在哪个分支

遇到不知道下一步做什么时，先运行：

```bash
git status
```

## 添加到暂存区

添加某个文件：

```bash
git add 文件名
```

添加当前目录下所有改动：

```bash
git add .
```

取消暂存：

```bash
git restore --staged .
```

初学时可以常用 `git add .`，但提交前一定要先看 `git status`，避免把不该提交的大文件、临时文件也加进去。

## 提交版本

```bash
git commit -m "提交说明"
```

例如：

```bash
git commit -m "添加 Git 学习笔记"
```

提交说明要写“这次做了什么”，不要写成：

```text
update
111
fix
```

更好的写法：

```text
添加 README 项目介绍
整理 Python 基础练习目录
忽略虚拟环境和模型文件
```

## 查看提交历史

```bash
git log
```

简洁版本：

```bash
git log --oneline
```

这个命令可以看到每一次 commit。

在 git log 的分页查看器中，可以：
```bash
q - 退出
空格键 - 向下翻一页
b - 向上翻一页
↑ / ↓ - 逐行上下滚动
/关键词 - 搜索内容（按 n 查找下一个）
g - 跳到开头
G - 跳到末尾
```

## 查看修改内容

查看工作区中还没有暂存的修改：

```bash
git diff
```

查看已经暂存、准备提交的修改：

```bash
git diff --cached
```

## 一次完整流程

```bash
git status
git add .
git status
git commit -m "添加 Git 学习笔记"
git log --oneline
```

这就是最基础的一轮 Git 操作。

## 本节练习

1. 新建一个 `hello.txt`
2. 写入一句话
3. 执行 `git status`
4. 执行 `git add hello.txt`
5. 再执行 `git status`
6. 执行 `git commit -m "添加 hello 文件"`
7. 执行 `git log --oneline`

## 初学者要记住

`git add` 不是提交。

`git commit` 才是真正保存一个版本。

## 补充：如何看懂 git status

`git status` 是 Git 初学阶段最重要的命令。它像一个仪表盘，告诉你当前仓库处于什么状态，以及下一步应该做什么。

初学时，不用害怕它输出的英文。你先重点看 4 类信息，下面每一项都可以展开查看。

- `Untracked files`
- `Changes not staged for commit`
- `Changes to be committed`
- `nothing to commit, working tree clean`

### 先记口诀

看到红色：

```text
还没准备好提交
```

通常下一步是：

```bash
git add 文件名
```

看到绿色：

```text
已经准备好提交
```

通常下一步是：

```bash
git commit -m "提交说明"
```

看到 clean：

```text
当前没有东西要提交
```

可以继续写代码，或者执行 `git push`。

<details>
<summary>1. Untracked files：新文件还没被 Git 管理</summary>

如果你看到：

```text
Untracked files:
  (use "git add <file>..." to include in what will be committed)
        hello.txt
```

意思是：`hello.txt` 是一个新文件，Git 还没有开始跟踪它。

这个状态通常是红色。

下一步一般是：

```bash
git add hello.txt
```

如果你想一次添加当前目录下所有改动：

```bash
git add .
```

</details>

<details>
<summary>2. Changes not staged for commit：文件改了，但还没放进暂存区</summary>

如果你看到：

```text
Changes not staged for commit:
  (use "git add <file>..." to update what will be committed)
        modified:   hello.txt
```

意思是：`hello.txt` 以前已经被 Git 管理过，现在你又修改了它，但是这次修改还没有加入暂存区。

这个状态通常也是红色。

如果你想提交这次修改，下一步是：

```bash
git add hello.txt
```

如果你想先看看具体改了什么：

```bash
git diff hello.txt
```

</details>

<details>
<summary>3. Changes to be committed：已经暂存，准备提交</summary>

如果你看到：

```text
Changes to be committed:
  (use "git restore --staged <file>..." to unstage)
        modified:   hello.txt
```

意思是：`hello.txt` 已经进入暂存区，下一次 `git commit` 会把它保存成版本记录。

这个状态通常是绿色。

下一步一般是：

```bash
git commit -m "修改 hello 文件"
```

如果你发现这个文件不想放进本次提交，可以取消暂存：

```bash
git restore --staged hello.txt
```

注意：取消暂存不会删除你的代码，只是把文件从“准备提交”状态拿出来。

</details>

<details>
<summary>4. nothing to commit, working tree clean：当前没有需要提交的内容</summary>

如果你看到：

```text
nothing to commit, working tree clean
```

意思是：当前工作区是干净的，没有新文件、没有未提交修改。

这通常说明：

- 你刚刚完成了一次 commit
- 或者你确实没有修改任何文件
- 当前本地仓库状态稳定

这时不需要再执行 `git add` 或 `git commit`。

如果你已经连接了 GitHub，并且本地有新 commit，可以考虑：

```bash
git push
```

</details>

<details>
<summary>5. 最常见的一次状态变化：从新建文件到提交完成</summary>

新建文件后：

```text
Untracked files
```

执行：

```bash
git add .
```

状态变成：

```text
Changes to be committed
```

执行：

```bash
git commit -m "添加新文件"
```

状态变成：

```text
nothing to commit, working tree clean
```

这就是 Git 最核心的流程：

```text
工作区 -> 暂存区 -> 本地仓库
```

</details>
