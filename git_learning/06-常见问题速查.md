# 06 常见问题速查

这一节用来查 Git 常见问题。

## 查看当前状态

不知道发生了什么时，先执行：

```bash
git status
```

这是最重要的排查命令。

## add 错文件了怎么办

如果文件已经 `git add`，但还没有 commit：

```bash
git restore --staged 文件名
```

这会把文件从暂存区拿出来，但不会删除你的修改。

## 修改了文件但想放弃修改

危险操作，确认不要这些修改后再执行：

```bash
git restore 文件名
```

它会把文件恢复到上一次 commit 的状态。

## commit 信息写错了

如果只是最近一次 commit 的说明写错：

```bash
git commit --amend -m "新的提交说明"
```

注意：如果这个 commit 已经 push 给别人，不要随便 amend，可能影响协作。

## commit 后发现漏了一个文件

先添加漏掉的文件：

```bash
git add 漏掉的文件
```

再合并到上一次提交：

```bash
git commit --amend
```

如果不想改上一次提交，也可以重新提交一次：

```bash
git commit -m "补充遗漏文件"
```

## 查看某个文件改了什么

```bash
git diff 文件名
```

如果已经 add：

```bash
git diff --cached 文件名
```

## 查看简洁提交历史

```bash
git log --oneline
```

## 查看当前绑定的 GitHub 仓库

```bash
git remote -v
```

## 修改远程仓库地址

```bash
git remote set-url origin 新地址
```

## 拉取远程代码前的建议

先看状态：

```bash
git status
```

如果当前有没提交的修改，建议先 commit 或临时保存，再 pull。

## 临时保存当前修改

```bash
git stash
```

恢复临时保存：

```bash
git stash pop
```

初学时不需要频繁使用 stash，但知道它存在就好。

## 最常用排查顺序

遇到 Git 问题时，按这个顺序看：

```bash
git status
git branch
git log --oneline
git remote -v
```

这几个命令通常能帮你判断：

- 当前在哪个分支
- 有没有未提交修改
- 最近提交了什么
- 有没有连接 GitHub

## 初学者要记住

Git 出问题时不要乱敲命令。

先看 `git status`，再判断下一步。
