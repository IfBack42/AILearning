# 跟着黑马学 AI

这是我的人工智能学习项目仓库，用来记录从 Python 基础、数据库、数据分析、机器学习到深度学习的学习过程。仓库里既有课堂代码，也有自己写的小练习、小项目、数据处理脚本和建模实验。

这个 README 不是最终版，而是一个适合放到 GitHub 仓库首页的示例。它的目标是让别人打开仓库后能快速知道：我学了什么、代码放在哪里、每个目录大概对应哪些知识点。

## 学习路线

目前仓库内容大致可以分成几个阶段：

1. Python 编程基础
2. Python 高级语法与小项目
3. Web 自动化与爬虫
4. Flask 服务端基础
5. SQL 数据库基础
6. Pandas、Matplotlib、文本分析等数据分析内容
7. 传统机器学习与集成学习
8. 深度学习、自动微分、神经网络、图像分类和目标检测
9. 数学建模、课程作业、比赛实践与综合项目

## 项目结构

```text
.
├── Python_study/              # Python 基础、高级语法、爬虫、Flask、小游戏等练习
├── 数据库/                    # SQL 基础语句练习，包括 DDL、DML、DQL
├── 数分/                      # 数据分析学习内容，包括 Pandas、Matplotlib、文本预处理等
├── 机器学习/                  # 机器学习与集成学习实验，如随机森林、GBDT、XGBoost

└── workworkwrokwrok/          # 课程作业、数学建模、LDA 主题模型、评论分析等综合实践
```

> 说明：仓库中包含一些数据集、模型文件、图片、压缩包和课程资料。上传 GitHub 时，建议根据文件大小和隐私情况决定是否保留，较大的数据文件可以考虑放到网盘或使用 Git LFS。

## 各目录学到了什么

### Python_study

这个目录主要记录 Python 编程学习过程，是整个仓库的基础部分。

已覆盖内容包括：

- Python 基础语法：变量、数据类型、输入输出、格式化字符串
- 流程控制：`if`、`for`、`while`、`while-else`
- 常用数据结构：字符串、列表、元组、字典、集合
- 函数：函数定义、参数、返回值、嵌套调用
- 异常处理：`try-except`、简单文件异常处理
- 文件读写：读取文本、统计内容、写入结果文件
- 面向对象：类、对象、类属性、对象方法
- 小项目练习：学生管理系统、简易计算器、点餐系统、银行系统、猜拳游戏等
- 高级语法：闭包、装饰器、时间统计装饰器、带参数装饰器
- Playwright：浏览器自动化、CSS 定位、角色定位、trace 跟踪、Steam 评论抓取
- Flask：简单服务端封装
- 马里奥项目：用 Python 组织一个更完整的游戏项目结构

代表性文件示例：

- `Python_study/初级编程阶段/学生管理系统.py`：练习列表、字典、函数拆分和简单增删改查逻辑。
- `Python_study/初级编程阶段/简易计算机.py`：练习条件判断、输入处理和基础运算。
- `Python_study/初级编程阶段/文件读取并计算平均分/`：练习文件读取、数据统计和结果写入。
- `Python_study/高级编程阶段/闭包&装饰器/`：理解函数可以作为对象传递，学习闭包和装饰器。
- `Python_study/Playwright/STEAM游戏评论抓取.py`：练习自动化打开网页、定位元素、抓取评论数据并保存。
- `Python_study/Flask封装/Flask服务端.py`：初步理解后端接口和 Web 服务。

## 如何运行代码

本仓库主要是学习代码，不是一个单一应用，因此不同文件的运行方式可能不同。

常见运行方式如下：

```bash
python 文件名.py
```

例如：

```bash
python Python_study/初级编程阶段/学生管理系统.py
```

如果是 Jupyter Notebook 文件，可以使用：

```bash
jupyter notebook
```

如果涉及机器学习、深度学习、爬虫或数据分析，可能需要先安装依赖，例如：

```bash
pip install pandas numpy matplotlib scikit-learn jieba playwright flask torch
```

Playwright 第一次使用时通常还需要安装浏览器：

```bash
playwright install
```

## 建议的 GitHub 上传整理

如果这个仓库要作为正式 GitHub 项目展示，建议后续逐步整理：

- 添加 `.gitignore`，忽略 `.venv/`、`.idea/`、`__pycache__/`、大型数据集和临时输出文件。
- 把学习代码、数据集、模型文件分开存放，例如 `src/`、`data/`、`models/`、`notebooks/`。
- 对重要脚本添加文件开头注释，说明这个文件练习了什么。
- 对比较完整的小项目单独写子 README，例如 Playwright 抓取项目、LDA 主题分析项目、机器学习红酒分类项目。
- 如果模型文件或数据集很大，不建议直接传到 GitHub，可以使用网盘、Git LFS 或只保留小样例数据。
